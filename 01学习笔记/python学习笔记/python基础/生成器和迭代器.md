# 迭代器 & 生成器 完整精讲

## 一、先理清关系

1. **迭代器 Iterator**：统一遍历规则的对象，是**底层规范**
    
2. **生成器 Generator**：**特殊的迭代器**，写法更简单，节省内存
    
3. 所有能被 `for` 循环遍历的对象，都叫**可迭代对象 Iterable**
    

---

# 1. 可迭代对象（Iterable）

### 定义

内部实现了 `__iter__()` 方法的对象，就是可迭代对象 常见：列表、元组、字符串、字典、集合、range

### 特点

- 能被 `for` 遍历
    
- 不能直接取值，必须转成迭代器
    

### 判断方法

```python
from collections.abc import Iterable

lst = [1,2,3]
print(isinstance(lst, Iterable))  # True
```

---

# 2. 迭代器（Iterator）

## 核心两个方法

1. `__iter__()`：返回自身迭代器
    
2. `__next__()`：**取出下一个数据**，没有数据抛 `StopIteration` 异常
    

## 作用

**惰性取值**：一次只拿一个数据，不一次性加载全部到内存

## 1）可迭代对象 → 转为迭代器

```python
lst = [10,20,30]
# 转迭代器
it = iter(lst)

# 手动取值
print(next(it))
print(next(it))
print(next(it))
# 再取报错 StopIteration
```

## 2）for 循环遍历迭代器原理

1. `for` 自动调用 `iter()` 转迭代器
    
2. 不断调用 `next()` 取值
    
3. 捕获 `StopIteration` 自动结束循环
    

## 3）自定义迭代器（手写）

自己写一个能遍历的迭代器

```python
class MyIterator:
    def __init__(self, start, end):
        self.start = start
        self.end = end

    def __iter__(self):
        return self

    def __next__(self):
        if self.start > self.end:
            raise StopIteration
        res = self.start
        self.start += 1
        return res

# 使用
obj = MyIterator(1,5)
for i in obj:
    print(i)
```

流程： `__iter__` 返回自己 → for 循环不断调用 `__next__` → 超出范围抛异常结束

# 3. 生成器 Generator（重点）

## 一句话

**生成器 = 简易迭代器，不用手写类，用 yield 实现** **优势：代码极简，极度省内存**

## 核心关键字：yield

- `return`：直接结束函数，返回值
    
- `yield`：**暂停函数，保存当前状态，返回值**，下次执行从暂停处继续
    

## 写法 1：生成器表达式（最简）

语法和列表推导式几乎一样，把 `[]` 改成 `()`

```python
# 列表推导式：一次性生成全部数据占内存
lst = [i*2 for i in range(5)]

# 生成器表达式：惰性生成，用一个取一个
gen = (i*2 for i in range(5))

print(next(gen))
print(next(gen))
```

## 写法 2：生成器函数（最常用）

函数里出现 `yield`，这个函数就是**生成器函数** 调用函数不会执行代码，只会创建生成器对象

```python
def gen_func():
    print("第一段代码")
    yield 1
    print("第二段代码")
    yield 2
    print("第三段代码")
    yield 3

# 创建生成器对象，代码不执行
g = gen_func()

# 第一次next，执行到第一个yield暂停
print(next(g))
# 第二次next，从暂停处继续到第二个yield
print(next(g))
# 第三次同理
print(next(g))
```

### 执行流程死记

1. 调用 `gen_func()` → 创建生成器，**不执行函数内部**
    
2. `next(g)` → 执行代码，遇到 `yield` 暂停，返回值
    
3. 再次 `next()` → 从上次暂停位置继续往下走
    

## for 循环遍历生成器

```python
def gen_num(n):
    for i in range(n):
        yield i

g = gen_num(4)
for num in g:
    print(num)
```

# 4. send () 给生成器传值（进阶）

`next()` 只能取值 `send(数据)` 既可以唤醒生成器，还能**向内部传参**

```python
def eat():
    food = yield
    print(f"吃：{food}")

g = eat()
next(g)  # 先启动生成器
g.send("米饭")
g.send("面条")
```

# 5. 迭代器 与 生成器 核心区别

1. **迭代器**
    
    1. 基于类实现，重写 `__iter__` `__next__`
        
    2. 代码繁琐，适合复杂遍历逻辑
        
    3. 手动控制遍历规则
        
2. **生成器**
    
    1. 基于 `yield` 实现，属于迭代器子类
        
    2. 代码极简，开发最快
        
    3. 自动保存状态，自动处理异常
        
    4. **内存无敌**：海量数据首选
        

## 共同特点

- 都是**惰性迭代**，边用边生成
    
- 数据只能**从头到尾遍历一次**，遍历完清空
    
- 都节省内存，适合大数据、文件读取、爬虫分批拿数据
    

---

# 6. 实战场景

1. 超大文件读取，一行一行读（防止一次性读崩内存）
    
2. 爬虫分批请求数据
    
3. 海量数字序列生成
    
4. 协程任务调度（yield 早期实现协程）
    

---

# 7. 超简总结背诵

1. 可迭代对象：能 for 循环，不一定能 next
    
2. 迭代器：有`__iter__`和`__next__`，能用 next 取值
    
3. 生成器：含 yield 的函数 / 表达式，是简化版迭代器
    
4. yield 暂停 return 结束
    
5. 大数据优先用生成器，内存最优
