## 回答

这道题我分两块来讲：模型怎么被训练出工具调用能力，以及训练好之后运行时是怎么工作的。

简单来说：**SFT 负责教模型“怎么调”（格式与语法），而 RLHF/DPO 负责教模型“何时调、调哪个，该不该调”（策略与决策与安全问题）。**

以下是这三种技术在工具调用（Tool Calling）中的具体作用与工程实践：

### 1. SFT-监督学习：内化业务核心范式（解决“说对话、会调用”）

SFT 是模型学会调工具的绝对基石。预训练模型本身并不懂什么是“API”或“JSON格式”，SFT 通过大量高质量的“问题-工具调用”样本，让模型完成以下能力：

- **掌握工具调用规则**：学习不同业务工具的调用条件、参数格式与返回结果的处理逻辑（例如 API 的参数校验、错误重试策略）。
- **理解领域术语与任务拆解**：学会将复杂的自然语言指令（如“帮我退款”）拆解为标准的工具调用步骤（如“订单校验 → 库存核对 → 退款申请”）。
- **严格的格式遵循**：让模型能够稳定地输出特定格式的代码或 JSON，以便下游系统能够正确解析和执行。

### 2. RLHF-强化学习：在线强化与复杂推理（解决“何时调、怎么调好”）

虽然 SFT 能让模型按格式输出，但它无法赋予模型真正的“判断力”。RLHF 通过引入奖励模型和在线反馈，弥补了这一短板：

- **学会动态决策**：RLHF 的目标是让模型学会“何时发起搜索、使用哪种工具、搜索什么内容，以及如何基于检索到的内容决定下一步行动（是直接给出答案，还是继续发起新一轮搜索）”。
- **自我反思与纠正**：通过在线强化学习，模型能够在多轮交互中根据环境的实时反馈（如工具返回的错误信息）进行自我反思，优化多步推理和工具调用的策略。
- **多目标平衡**：在复杂的业务场景中，RLHF 能够平衡“准确性”与“效率”、“安全性”与“覆盖面”之间的冲突，使模型在约束条件下做出最优的工具调用决策。

### 3. DPO-偏好对齐：轻量级的策略对齐（解决“调优成本与偏好对齐”）

作为 RLHF 的轻量级替代方案，DPO 在工具调用场景中同样发挥着关键作用，尤其适合资源受限或需要快速迭代的工程：

- **对比学习优化决策**：DPO 通过给模型同时展示“正确的工具调用轨迹（偏好回答）”和“错误的/低效的调用轨迹（非偏好回答）”，引导模型直接学习更优的调用策略。
- **低成本策略迭代**：在对话系统或 Agent 需要持续迭代的场景中，DPO 能够以更低的计算成本和更稳定的训练过程，快速适应用户对工具调用偏好的变化。

### 总结：工程中的融合架构

在实际的 Agent 开发中，这三种技术通常是协同工作的：

1. 先通过 **SFT** 让模型掌握基础的 API 调用格式和业务规则。
2. 再通过 **RLHF** 或 **DPO** 对模型的调用策略进行对齐，使其具备在复杂、动态环境下的自主决策能力。

这种“范式内化（SFT）+ 策略对齐（RLHF/DPO）”的融合架构，是当前打造高质量、高可用 AI Agent 的核心路径。

在大语言模型（LLM）的训练流程中，**SFT（Supervised Fine-Tuning，监督微调）** 是承上启下的核心阶段。如果说预训练是让模型“博览群书”，那么SFT就是给模型上“规范特训”，教会它听懂人类指令、按格式输出、正经回答问题。

基于当前的技术现实与工程实践，以下是关于SFT的全面解析：

### 1. 为什么需要 SFT？

经过预训练（Pre-training）的裸模型虽然拥有海量知识，但本质上只是在“做词语接龙”，听不懂人类指令，回答随意发散且没有安全底线。SFT 的核心目标就是解决这些问题，赋予模型三大关键能力：

- **指令遵循能力（Instruction Following）**：让模型能够理解并严格按照用户的Prompt行事（例如输入“请总结以下内容”，模型必须按要求总结，而不是随意发挥）。
- **人类风格对齐（Human Alignment）**：规范模型的输出格式和风格，使其具备多轮对话能力，输出结构规整、逻辑通顺。
- **安全性（Safety）**：通过数据让模型学习安全拒答策略，避免生成非法、有害或违规的内容。

### 2. SFT 的核心原理与流程

SFT 的本质是基于“下一个Token预测”目标，使用高质量的人工标注数据进行有监督学习。

- **数据准备**：通常采用“指令-输入-输出”或“多轮对话”的结构化格式（如 Alpaca 或 ShareGPT 格式）。
- **模型初始化**：基于预训练模型（如 LLaMA、Qwen 等）加载权重，保留其通用知识。
- **损失函数**：采用标准的**交叉熵损失函数（Cross-Entropy Loss）**，让模型模仿标准优质回答，最大化对参考答案的似然概率。

### 3. 主流工程实践：参数高效微调（PEFT）

在实际工程中，由于全参数微调计算资源需求极大，目前业界广泛采用参数高效微调技术，其中 **LoRA** 和 **QLoRA** 是绝对的主流：

- **LoRA（Low-Rank Adaptation）**：冻结预训练模型的主干参数，仅在参数矩阵旁新增由低秩矩阵构成的旁路进行训练。这大幅降低了计算开销，且在小数据量场景下能有效防止过拟合。
- **QLoRA（Quantized LoRA）**：在LoRA的基础上，将大模型量化为 4bit 或 8bit 加载，进一步降低了显存消耗，使得在消费级显卡上微调大模型成为可能。

### 4. SFT 的局限性

尽管SFT让模型学会了“怎么回答”，但它存在天生的局限性：

- **无法识别恶意诱导**：SFT只能模仿样本，如果坏人层层套话、伪装正常提问，纯SFT模型很容易沦陷，泄露敏感信息。
- **缺乏审美与价值观判断**：SFT教不会模型“什么该答、什么不该答、怎么答更好”。

因此，在完整的LLM训练三阶段中，SFT 之后通常还需要进行 **RLHF（基于人类反馈的强化学习）** 或 **DPO（直接偏好优化）** 等对齐训练，才能最终给模型装上价值观与安全锁，使其成为懂分寸、守规矩的成熟AI助手。

## 概念2 RLHF 和DPO

如果说 SFT 是教模型“怎么说话”，那么 RLHF 和 DPO 就是教模型“怎么把话说好”以及“什么话绝对不能说”。它们属于大模型对齐（Alignment）阶段的进阶技术。

基于当前的工程实践和技术演进，以下是 RLHF 与 DPO 的深度解析与对比：

### 1. RLHF：经典的对齐基石

**RLHF（Reinforcement Learning from Human Feedback，基于人类反馈的强化学习）** 是 ChatGPT 早期爆火的核心技术，它通过引入强化学习让模型适应人类的偏好。

- **核心流程（三阶段）**：
    1. **SFT**：先让模型学会基础对话。
    2. **训练奖励模型（Reward Model, RM）**：让模型对同一个问题生成多个回答，由人类标注员进行排序（Rank）。然后用这些排序数据训练一个能“打分”的 RM。
    3. **PPO 强化学习**：将 SFT 模型作为策略（Policy），RM 作为环境反馈，使用 PPO 算法不断调整模型参数，使其输出能获得更高的奖励分数。同时引入 KL 散度惩罚，防止模型为了拿高分而“胡言乱语”。
- **优点**：理论上限极高，能捕捉到人类非常微妙甚至难以言表的偏好。
- **痛点**：工程极其复杂。需要同时加载 4 个模型（Actor, Critic, Reference, Reward），显存开销巨大；且 PPO 算法对超参数极度敏感，训练容易崩溃。

### 2. DPO：当前工程界的主流新宠

**DPO（Direct Preference Optimization，直接偏好优化）** 是 2023 年底提出、并在 2024-2026 年间彻底引爆工程界的技术。它的核心思想是：**既然 RLHF 那么麻烦，我们能不能跳过奖励模型，直接用偏好数据优化语言模型？**

- **核心原理**： DPO 在数学上推导出了 RLHF 目标函数的最优解，将其转化为一个极其简单的**分类损失函数（类似交叉熵）**。它不再训练 RM，而是直接拿“人类喜欢的回答”和“人类讨厌的回答”成对的数据，去更新 SFT 模型的参数。
- **优点**：
    - **极度轻量**：训练时只需加载一个模型，显存需求大幅降低，训练速度比 RLHF 快数倍。
    - **极其稳定**：抛弃了复杂的 PPO 算法，训练过程像 SFT 一样稳定，几乎不会崩溃。
    - **效果持平甚至更优**：在大量开源模型（如 LLaMA-3, Qwen 系列）的官方对齐中，DPO 展现出了不输甚至超越 RLHF 的效果。

### 3. 2026年视角下的工程实践现状

结合当下的技术现实，业界在模型对齐阶段的选择已经非常明确：

1. **DPO 成为绝对主流**：目前绝大多数开源大模型的官方对齐版本，以及企业内部的微调工程，**首选都是 DPO**（或其变体如 KTO, ORPO, SimPO）。它性价比极高，是目前的“版本答案”。
2. **RLHF 退居二线或特定场景**：传统的 PPO 流程因为成本太高，已经很少被从头训练。但在某些需要极致对齐、或者需要利用已有高质量 RM 进行持续学习的场景下，RLHF 依然有其价值。
3. **数据质量 > 算法选择**：工程实践反复证明，无论是 RLHF 还是 DPO，**偏好数据的质量远比算法本身重要**。几千条由专家精标的“好/坏”对比数据，效果往往吊打几万条由普通用户随手点的劣质数据。

**总结来说：** SFT 给了模型“智商”，RLHF 证明了模型可以“对齐人类”，而 DPO 则是用一种极其优雅、高效的数学方式，让“对齐人类”这件事在工程上变得触手可及。如果你现在要微调一个模型，**SFT + DPO** 是最推荐的黄金组合。
