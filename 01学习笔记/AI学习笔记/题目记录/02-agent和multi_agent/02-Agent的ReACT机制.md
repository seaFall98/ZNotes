
在 LangChain 中，Agent 的 React 模式 (Reasoning + Acting, 推理-行动)：通过动态循环实现复杂任务的自主解决。该模式将大语言模型 (LLM) 的推理能力与外部工具的执行能力结合，形成“思考-行动-观察”的迭代流程。

React 模式的本质是“推理驱动行动，行动反馈推理”的循环，其核心步骤可概括为以下四步：

- **思考 (Thought)**：LLM 分析用户输入的任务，结合已有知识和工具描述，推理出需要采取的行动（如是否需要调用工具、调用哪个工具）。
- **行动 (Action)**：根据推理结果，选择合适的工具（如搜索、计算、文件操作）并执行，生成工具输出。
- **观察 (Observation)**：将工具输出反馈给 LLM，作为下一步推理的依据。
- **迭代 (Iteration)**：重复“思考-行动-观察”循环，直到 LLM 判断任务完成，生成最终答案。

在 langchain 框架有一个对象，**AgentExecutor** 是 React 模式的运行控制器，负责管理循环流程（思考→行动→观察），协调 LLM 与工具的交互。
