可以这样回答，重点不是“继续重试”，而是**把失败纳入 Agent 的控制流和降级策略**。

## 面试回答版

智能体调用工具失败超过 3 次后，我一般不会继续无脑重试，而是把它判定为**不可恢复失败**，进入降级处理流程。

核心处理思路是：

1. **停止重试，避免放大故障**
    
    - 超过最大重试次数后直接熔断；
        
    - 避免重复调用外部 API、数据库、支付接口等造成资源浪费或副作用。
        
2. **区分失败类型**
    
    - 如果是临时性错误，比如网络超时、限流、服务 5xx，可以短暂重试；
        
    - 如果是参数错误、权限错误、工具 schema 不匹配、业务校验失败，继续重试没有意义，应立即失败；
        
    - 如果工具有副作用，比如下单、发邮件、扣款，要特别小心，必须保证幂等。
        
3. **把失败结果反馈给 Agent**
    
    - 将工具失败原因、错误码、尝试次数、最后一次错误信息返回给模型；
        
    - 让 Agent 判断是否可以换工具、调整参数、走备用方案，或者向用户澄清。
        
4. **执行降级方案**
    
    - 有备用工具就切换备用工具；
        
    - 有缓存就返回缓存结果；
        
    - 能人工介入就转人工；
        
    - 如果无法完成，就给用户明确说明失败原因和可选操作。
        
5. **记录日志和告警**
    
    - 记录 traceId、tool name、输入参数摘要、错误类型、重试次数、耗时；
        
    - 对高频失败的工具触发监控告警；
        
    - 后续用于排查工具稳定性、Prompt 参数生成问题、权限问题或外部服务故障。
        

## 可以直接背的版本

> 如果 Agent 调用某个工具重试 3 次以上仍然失败，我不会继续让它无限重试，而是会触发最大重试保护，把这次工具调用标记为失败，并进入降级流程。
> 
> 首先要区分失败类型：如果是网络超时、限流、临时 5xx，可以用指数退避重试；如果是参数错误、权限错误、schema 不匹配，继续重试没有意义，应直接失败。
> 
> 超过重试上限后，我会把错误信息结构化返回给 Agent，让它判断是否可以换备用工具、重新规划任务、使用缓存结果，或者向用户请求补充信息。如果这个工具涉及发邮件、下单、支付等有副作用操作，还要依赖幂等 key 防止重复执行。
> 
> 最后还要记录日志、traceId、错误码、重试次数，并接入监控告警。这样可以避免 Agent 陷入死循环，也能保证系统稳定性和可观测性。

## 工程上可以设计成这样

```java
public ToolResult callToolWithRetry(ToolRequest request) {
    int maxRetries = 3;

    for (int attempt = 1; attempt <= maxRetries; attempt++) {
        try {
            return toolClient.call(request);
        } catch (TransientToolException e) {
            // 临时性错误：网络超时、限流、服务 5xx，可以重试
            sleepWithBackoff(attempt);
        } catch (NonRetryableToolException e) {
            // 非重试型错误：参数错误、权限错误、schema 错误，直接失败
            return ToolResult.failed("NON_RETRYABLE_ERROR", e.getMessage());
        }
    }

    // 超过最大重试次数，进入降级或交给 Agent 重新规划
    return ToolResult.failed(
        "TOOL_RETRY_EXCEEDED",
        "Tool failed after 3 retries. Please replan or use fallback."
    );
}
```

## 面试官想听到的关键词

|关键词|含义|
|---|---|
|最大重试次数|防止 Agent 无限循环|
|指数退避|避免瞬时放大流量|
|错误分类|区分可重试和不可重试错误|
|幂等性|防止重复执行有副作用操作|
|降级方案|备用工具、缓存、人工介入|
|Agent replan|让 Agent 重新规划任务|
|熔断|工具持续失败时临时禁用|
|可观测性|日志、traceId、指标、告警|

## 一句话总结

**工具重试 3 次以上仍失败，应该停止重试，分类错误，结构化返回失败信息，让 Agent 重新规划或走降级方案，同时做好幂等、熔断、日志和告警。**
