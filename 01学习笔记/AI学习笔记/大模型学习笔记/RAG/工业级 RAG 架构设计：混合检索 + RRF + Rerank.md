# 工业级 RAG 架构设计
[# 工业级 RAG 架构设计：混合检索 + RRF + Rerank](https://github.com/Hao-yiwen/yiwen-blog-website/blob/master/docs/backend/architecture/hybrid-rag-rerank.md)

工业级 RAG 系统的核心设计理念：

```
多路召回 (高召回率) → RRF 融合 (公平合并) → Cross-Encoder 重排 (高精度)
```

这套架构在保证召回率的同时，通过重排序大幅提升了最终结果的相关性，是当前业界验证有效的主流方案。
# 多路召回有哪些路径？
在现代RAG系统里，多路召回早已超越了传统的“全文+语义”组合，形成了一个立体的检索网络。

我把它们分成四个层面，帮你建立一个完整的图景：

### 🧩 多路召回的四大路径

**1. 文本匹配**
这是最经典的基础路径。

-   **全文检索 (BM25)**：负责**字面匹配**，确保含有精确关键词的文档不会漏掉。
-   **语义检索 (密集向量/ANN)**：负责**语义匹配**，能找出意思相近但用词不同的文档，是处理长尾、模糊查询的关键。

**2. 结构关系
当数据间存在复杂的关系网络时，这条路极其有效。

-   **图检索 (如Neo4j)**：利用知识图谱，顺着实体间的关系路径进行检索。例如，查询“A公司的竞争对手”，可以通过图谱的关系边直接找到答案。
-   **关系检索**：在多表数据库中，通过外键关联来查询跨表信息，适用于结构化数据分析。

**3. 稀疏学习路径**
这是一条前沿且高效的路线，旨在用神经网络生成可解释的稀疏向量，兼具关键词匹配和语义理解的优点。

-   **SPLADE / BGE-M3**：这些模型能将查询和文档转化为高维稀疏向量。它生成的不是无意义的数字，而是带有权重的“关键词”集合（如 `{"苹果": 0.8, "手机": 0.7}`），非常适合与BM25共享倒排索引架构。

**4. 元数据与规则路径**
这条路利用数据和业务本身的结构化信息，简单直接。

-   **过滤检索**：根据时间、类别、作者等字段进行预过滤，缩小检索范围。
-   **业务规则召回**：如“优先展示VIP用户的内容”、“最新3天内发布的文档”等业务硬规则。

### 🎯 总结：动态的“分子料理”策略

一个成熟的RAG系统，召回策略往往是动态的，会根据查询意图自动组合最优路径。例如：

-   **精确查询**（如产品型号“ABC-123”）：**全文检索** 优先。
-   **模糊咨询**（如“如何缓解头痛”）：**语义检索** 主导。
-   **关系查询**（如“A公司供应商的客户”）：**图检索** 是关键。
-   **冷门术语**（如专业缩写“NLP”）：**SPLADE** 可能比纯语义检索表现更好。

所以，理解可以进一步扩展为：**全文检索、语义检索、图检索、学习型稀疏检索、元数据过滤**等多条路径并行，然后由**RRF**进行初步融合，最后再由精排模型做最终排序。
